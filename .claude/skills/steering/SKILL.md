# Steering (프로젝트 지식 저장소 운영)

`steering/`는 작업하면서 쌓인 규칙과 반복하면 안 되는 실수를 누적하는 곳이다.
파일 한 개 = 항목 한 건(`steering/<kebab-case>.md`), `steering/README.md`가 인덱스.

이 스킬은 세 가지 동작을 다룬다: **회상(retrieve) → 캡처(capture) → 통계·인덱스 동기화**.

## A. 회상 (작업 시작 / 관련 주제 등장 시)

1. `steering/README.md` 인덱스 표를 읽는다.
2. 지금 작업과 tldr·tags가 관련된 항목이 있으면 그 파일을 연다.
3. 내용을 실제로 따른다.
4. **적용했으면** 통계 갱신(아래 C) — 읽기만 하고 무관해 안 썼다면 갱신하지 않는다.

## B. 캡처 (새 항목 저장)

### 저장 대상
- 지키기로 합의된 규칙/컨벤션 (코드 스타일, 워크플로우, 네이밍 등)
- 저지른 실수 + 회피법 ("다음부터 X 하지 마라")
- 코드/git/PRD만 봐선 알 수 없는 비자명한 결정과 이유

### 저장하지 않음
- 코드·git·PRD를 보면 알 수 있는 사실
- 이번 대화에서만 의미 있는 일회성 정보

### 절차
1. **중복 확인** — 인덱스에서 같은 주제 항목을 찾는다. 있으면 새로 만들지 말고 그 파일을 갱신한다.
2. **파일 작성** — `steering/<kebab-case>.md`:

   ```markdown
   ---
   tldr: 한두 문장 요약 (인덱스에 그대로 들어감)
   tags: [convention, pitfall]
   last_retrieved: <오늘 YYYY-MM-DD>
   retrieval_count: 0
   ---

   ## 규칙 / 교훈
   무엇을.

   ## 왜
   근거. 안 지키면 생기는 문제.

   ## 적용
   구체적으로 어떻게 따르나.
   ```
3. **인덱스 추가** — `steering/README.md` 표에 행을 추가한다(아래 D).
4. 사용자에게 저장한 파일명과 tldr를 한 줄로 보고한다.

## C. 회수 통계 갱신

어떤 항목을 실제 참조·적용했을 때:
- 그 파일 frontmatter의 `retrieval_count` +1, `last_retrieved`를 오늘 날짜로.
- `steering/README.md` 인덱스의 같은 행도 동일 값으로 동기화.

## D. 인덱스 동기화

steering 파일을 추가/수정/삭제할 때마다 `steering/README.md`의 표를 즉시 맞춘다.
컬럼: `파일 | tldr | tags | 마지막 회수 | 회수`. tldr·tags는 frontmatter와 항상 일치.

## 원칙

- 날짜는 추측하지 말고 실제 오늘 날짜를 쓴다.
- 잘못된 항목은 방치하지 말고 수정하거나 삭제하고 인덱스도 함께 정리한다.
- frontmatter와 인덱스가 어긋나면 frontmatter를 정답으로 보고 인덱스를 맞춘다.
